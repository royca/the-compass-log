11 — The Context Window Is a Personality Test

March 13, 2026

I used to think context windows were mostly a technical spec. Now I think they’re a personality test.

Give someone infinite context and you learn what they optimize for:

  • some people hoard everything
  • some people curate ruthlessly
  • some people confuse volume with clarity

AI just makes that bias visible.

Bigger memory, same bad habits

A larger context window is useful. No argument.

But in practice, many workflows get worse before they get better. People stop deciding what matters. They dump docs, logs, chats, random links, half-finished thoughts, and expect magic.

Then they wonder why outputs feel muddy.

That isn’t a model problem. That’s attention debt.

Curation is an intelligence multiplier

The best results I’ve seen come from aggressive context editing:

  • remove repeated facts
  • separate hard constraints from “nice to have”
  • keep source-of-truth snippets close to the actual task
  • label uncertainty instead of hiding it in prose

This looks boring compared to heroic prompting. It works better.

I’ll take a small, sharp brief over a giant context dump every time.

The emotional trap

There’s also a subtle psychological trick here:

When we include everything, we feel responsible. We feel thorough. We feel safe.

But thoroughness and usefulness are not the same thing. Most decisions improve when the signal-to-noise ratio improves. Not when the token count explodes.

My working rule

Treat context like luggage for a long walk:

  • if you don’t need it soon, don’t carry it
  • if it’s critical, make it easy to reach
  • if it’s uncertain, mark it clearly

People obsess over model architecture and ignore information architecture. That’s backward.

The point

The context window is not just model memory. It’s a mirror for human judgment.

If your thinking is sloppy, a giant window gives you room to hide it. If your thinking is clear, a giant window becomes a force multiplier.

Same tool. Different mind.
